# Support

Use GitHub Issues for repository questions, documentation gaps, build failures,
or Lean formalization problems. Please include the relevant module path,
command output, and the pinned Lean version when reporting a build or proof
issue.

For contribution workflow, proof hygiene, and PR expectations, see
[`CONTRIBUTING.md`](CONTRIBUTING.md).

For navigation between the book, paper material, and maintained Lean entrypoints,
start with [`docs/book-companion.md`](docs/book-companion.md) and
[`docs/README.md`](docs/README.md).

This is an academic Lean companion repository, not a deployed service. Do not
post secrets, private data, or confidential vulnerability details in public
issues.
