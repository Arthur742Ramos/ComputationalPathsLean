- Path/Basic/Congruence.lean, Path/Basic/Context.lean (referenced by Step/RwEq)
- Path/Rewrite/*.lean (Step, Rw, RwEq, Quot, Normalization, PathExpr, PathTactic, etc.)

## (1) Operations the framework provides

### Core path/step operations (Core.lean)
- Step structure: src/tgt/proof with Step.symm and Step.map.
- Path structure: steps + proof with toEq, refl, ofEq, trans, symm, invA/cmpA.
- Cast/transport/subst: cast, transport, subst, transport2, subst2 with simp lemmas.
- Congruence: congrArg for unary functions.

### Congruence and contexts (Congruence.lean, Context.lean)
- mapLeft/mapRight/map2 for binary congruence, plus trans/symm/refl lemmas.
- Product/sigma helpers: prodMk, fst, snd, and transport/subst lemmas on products.
- Context and BiContext: map, substLeft/substRight, mapLeft/mapRight/map2, fixLeft/fixRight.
- DepContext helpers for dependent contexts (transportContext etc.).

### Rewrite system (Rewrite/*.lean)
- Step relation (Step.lean) with ~56 constructors: unit/inverse/assoc, map2_subst, product/sigma/sum/function, transport, context and congruence rules.
- Rw: reflexive/transitive closure, rw_trans, rw_of_step, rw_congrArg_*, rw_mapLeft/Right/2, context substitution rewrites.
- RwEq: symmetric closure, rweq_refl/symm/trans, rweq_trans_congr_left/right, rweq_trans_congr, rweq_congrArg_of_rweq, rweq_context_map_of_rweq, rweq_mapLeft/Right.
- Normalization: IsNormal, normalize, NormalForm, normalizeForm.
- PathExpr: expression syntax + eval, minimal Step/Rw/RwPlus, complexity/termination scaffolding, Join + local confluence interface.
- PathTactic: path_rfl, path_symm, path_trans/path_trans_via, path_congr_left/right, path_congr, path_assoc, path_unit_left/right, path_cancel_left/right, path_inv_distr, path_congrArg, path_simp, path_auto, path_decide.

### Quotients and homotopy operations (Quot + Homotopy)
- PathRwQuot: refl/symm/trans/ofEq/toEq, cmpA/invA, groupoid laws in the quotient.
- LoopSpace/LoopQuot: id/comp/inv, pow/zpow, cancellation and inverse-of-comp lemmas.
- FundamentalGroup (pi_1): id/mul/inv and group law lemmas via LoopQuot.
- FundamentalGroupoid: PathRwQuot-based groupoid with conjugation maps.
- HigherHomotopyGroups: PiN operations (piN_mul/one/inv) with Path witnesses.
- EckmannHilton: whiskerLeftRw/whiskerRightRw, hcompRw, interchange, OmegaTwo vcomp/hcomp.

## (3) Underused capabilities (with evidence)

Armada wave files inspected: EHPSequence.lean, GaneaFibration.lean, LSCategory.lean, NilpotenceTheorem.lean, QuasiCategory.lean. These primarily use Path.refl/Path.ofEq and placeholder True/Exists; no RwEq lemmas or path tactics.

Repo-wide checks in Path/Homotopy:
- mapLeft/mapRight/map2 appear only in HigherProductHomotopy.lean and ProductFundamentalGroup.lean (rg mapLeft|mapRight|map2).
- path_* tactics appear only in LoopIteration.lean (rg path_auto/path_simp/etc.).
- Context/BiContext/RewriteLift have no matches in Homotopy (rg Context|BiContext|RewriteLift).
- PathExpr is unused in Homotopy (rg PathExpr).
- Normalization is referenced only in ThomSpectra.lean and CobordismComputation.lean (rg normalize/normalizeForm).
- Whiskering/hcomp at the RwEq level (EckmannHilton) is isolated to that module.

Net effect: the rewrite engine (Rw/RwEq congruence, context substitution), path tactics, PathExpr/normalization, and whiskering/hcomp are mostly unused in armada files; armada modules often stop at Path.refl or Path.ofEq.

## (4) Patterns armada files should use to leverage the framework
